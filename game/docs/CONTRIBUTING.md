# Contributing Guidelines

How to Contribute
- Fork or branch from `main`; create `feature/*` or `fix/*`
- Keep changes small and testable

Before Submitting
- Run unit tests and manual smoke tests on desktop and mobile
- Verify performance budgets and accessibility checks

Pull Request Checklist
- Clear description and screenshots/GIFs of changes
- Tests included or updated
- Docs updated if behavior changes

Coding Standards
- Use TypeScript when adding logic; keep files modular
- Prefer vanilla DOM APIs; avoid heavy dependencies

Design Standards
- Respect brand palette and typography
- Keep UI uncluttered; prioritize thumb-friendly controls

Communication
- Use issues for discussions; label appropriately; be respectful and constructive
